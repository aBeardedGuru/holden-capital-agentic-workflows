# AGENTS.md - docs Node Access Rules

This node contains the human-readable source of truth for repo purpose, workflow design, and finance automation behavior.

## Scope

- workflow design docs
- business/product docs
- implementation guidance
- repo operating docs

## Rules

- Follow `docs/enterprise-development-standards.md` as the default documentation governance baseline.
- Keep docs aligned with actual schemas, scripts, and workflows.
- Prefer updating existing docs over creating overlapping docs.
- Write for future agents first: explicit assumptions beat narrative prose.
- If implementation behavior changes, update docs in the same change.

## Typical Work

- refine finance workflow specs
- document operating contracts
- document review and audit behavior
- explain why a workflow exists and how it should be validated
