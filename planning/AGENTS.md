# AGENTS.md - planning Node Access Rules

This node contains planning artifacts that define what should be built.

## Scope

- PRD
- sharded epics
- future story support docs

## Rules

- Keep planning artifacts implementation-oriented.
- Prefer one epic per file and self-contained GitHub issue bodies.
- Do not let planning sprawl into implementation contracts unless it is needed for execution.

## Simplicity Rule

Planning should answer:

1. what is being built
2. why it matters
3. what the next implementable unit is

If it answers much more than that, it is probably too large.
